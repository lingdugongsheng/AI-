多代理智能客服系统

📖 项目简介

多代理智能客服系统是一个基于 LangChain 和 LangGraph 构建的高级智能客服解决方案。系统采用多代理架构，通过意图识别、专业领域处理和质量评估，提供精准、专业的客户服务体验。

本项目整合了意图分类器、多领域专业Agent、质量检查器和对话管理系统，能够自动处理技术问题、订单查询、产品咨询等多种客服场景，并在需要时智能升级到人工客服。

✨ 核心功能

🎯 多代理架构

系统包含三个专业领域Agent，每个Agent专注于特定业务领域：

技术支持Agent (TechSupportAgent)
处理范围：技术问题、故障排除、使用帮助
核心工具：search_faq - 智能FAQ搜索
能力：
  分析用户遇到的技术问题
  提供清晰的故障排除步骤
  从FAQ数据库查找相关解决方案
  支持多步骤问题解决

订单服务Agent (OrderServiceAgent)
处理范围：订单查询、物流跟踪、退换货
核心工具：
  query_order - 订单信息查询
  track_shipping - 物流状态跟踪
能力：
  实时查询订单状态
  提供详细物流信息
  解答退换货相关问题
  主动提供相关信息

产品咨询Agent (ProductConsultAgent)
处理范围：产品咨询、价格询问、功能介绍、产品推荐
核心工具：
  search_product - 产品搜索
  get_product_recommendations - 智能推荐
能力：
  介绍产品功能和特点
  根据用户需求推荐合适产品
  解答价格和库存问题
  提供产品对比分析

🧠 意图识别系统

IntentClassifier 负责识别用户消息的业务意图：

tech_support：技术问题、故障排除、使用帮助
order_service：订单查询、物流跟踪、退换货
product_consult：产品咨询、价格询问、功能介绍
escalate：投诉、无法理解、需要人工

特点：
基于LLM的智能意图分类
返回置信度评分（0-1）
支持低置信度自动升级
提供分类原因说明

🔍 工具集成系统

系统集成了5个专业工具，支持Agent调用：
工具名称   功能描述   使用场景
query_order   查询订单信息   订单状态、产品详情、价格等

track_shipping   查询物流信息   物流单号跟踪、配送状态

search_product   搜索产品信息   产品功能、价格、库存查询

get_product_recommendations   产品推荐   根据预算和类别推荐产品

search_faq   常见问题搜索   技术问题、使用帮助

📊 质量评估系统

QualityChecker 评估客服回复质量，确保服务质量：

评估维度：
相关性（0-25分）：回复是否针对用户问题
完整性（0-25分）：是否提供了足够的信息
专业性（0-25分）：语言是否专业得体
有用性（0-25分）：是否真正帮助到用户

功能：
自动生成质量评分（0-100分）
判断是否需要升级到人工客服
提供评估说明
支持自动升级决策

🔄 工作流引擎

基于 LangGraph 构建的状态化工作流：

用户消息
    ↓
意图分类
    ↓
路由决策
    ↓
专业Agent处理
    ↓
质量检查
    ↓
升级判断
    ↓
最终响应

流程特点：
状态化管理：完整的对话状态跟踪
条件路由：根据意图和置信度智能路由
质量保障：每条回复都经过质量评估
智能升级：低质量或低置信度自动升级

💬 对话管理系统

自动历史维护：记录用户和助手的对话
上下文理解：基于对话历史理解用户意图
历史查看：提供API查询对话历史
历史清除：支持清除对话历史
历史截断：仅使用最近6轮对话，优化性能

📈 统计分析系统

实时记录和分析系统运行数据：

总查询次数：系统处理的总消息数
意图分布统计：按意图类型分类的查询次数
升级统计：升级到人工客服的次数
平均置信度：意图识别的平均置信度
平均质量评分：回复质量的平均评分
运行时长：系统持续运行时间
最后查询时间：最近一次查询时间
